# HTTP и WSGI с нуля

Этот документ объясняет низкоуровневые детали из `server.py`.
Если ученик еще не понимает HTTP-запрос и HTTP-ответ, сначала лучше прочитать `docs/http-data-exchange.md`.

Он рассчитан на ученика, который уже знает только одну главную идею:

```text
в HTTP есть клиент и сервер
```

Клиент в нашем проекте - браузер. Сервер - Python-программа `server.py`.

## 1. Что делает клиент

Клиент начинает обмен.

Например игрок открывает сайт:

```text
https://питоны-батоны.рф/
```

Браузер отправляет серверу HTTP-запрос:

```text
GET /
```

Это означает:

```text
сервер, отдай мне главную страницу
```

Когда игрок нажимает кнопку "Старт", браузер отправляет другой запрос:

```text
POST /api/start
```

Это означает:

```text
сервер, я отправляю тебе данные, начни игру
```

## 2. Что есть внутри HTTP-запроса

У HTTP-запроса есть несколько важных частей.

| Часть | Пример | Зачем нужна |
| --- | --- | --- |
| Метод | `GET`, `POST` | Показывает тип действия |
| Путь | `/`, `/api/start` | Показывает, к какой части сервера обращаются |
| Заголовки | `Content-Type: application/json` | Передают служебную информацию |
| Тело | `{"name": "Аня"}` | Передает основные данные, если они есть |

Запрос главной страницы обычно выглядит по смыслу так:

```text
GET /
```

Запрос старта игры выглядит по смыслу так:

```text
POST /api/start
Content-Type: application/json

{"name": "Аня"}
```

Пустая строка перед JSON важна: она отделяет заголовки от тела запроса.

## 3. Что сервер должен вернуть

Сервер отвечает HTTP-ответом.

У ответа тоже есть части:

| Часть | Пример | Зачем нужна |
| --- | --- | --- |
| Статус | `200 OK` | Сообщает, получилось или нет |
| Заголовки | `Content-Type: text/html` | Объясняют, что лежит в ответе |
| Тело | HTML, JSON или текст | Основное содержимое ответа |

Если все хорошо, сервер часто возвращает:

```text
200 OK
```

Если страница не найдена:

```text
404 Not Found
```

Если браузер отправил неправильные данные:

```text
400 Bad Request
```

## 4. Что такое заголовки

Заголовки - это служебные строки HTTP.

Например:

```text
Content-Type: text/html; charset=utf-8
```

Это значит:

```text
в теле ответа лежит HTML-текст в кодировке UTF-8
```

Другой пример:

```text
Content-Type: application/json; charset=utf-8
```

Это значит:

```text
в теле ответа лежит JSON в кодировке UTF-8
```

В `server.py` заголовки создаются здесь:

```python
headers = [
    ("Content-Type", content_type),
    ("Content-Length", str(len(body))),
    ("Cache-Control", "no-store"),
]
```

`Content-Length` сообщает браузеру, сколько байтов в теле ответа.

## 5. Почему в коде есть байты

По сети компьютеры передают не "строки Python", а байты.

Строка Python:

```python
"Привет"
```

Для отправки по сети превращается в байты:

```python
body = text.encode("utf-8")
```

Когда сервер получает байты от браузера, он превращает их обратно в строку:

```python
raw_body = body_bytes.decode("utf-8")
```

Поэтому в низкоуровневом сервере есть ручные `encode` и `decode`.

FastAPI-ветка прячет это внутри библиотеки. Основная ветка показывает это явно.

## 6. Что такое WSGI

Python сам по себе не знает, как принимать HTTP-запросы из браузера.

Нужен слой между интернетом и нашей функцией Python.

В этом проекте такой слой - `wsgiref`.

```python
from wsgiref.simple_server import make_server
```

`wsgiref` входит в стандартную библиотеку Python. Его не нужно устанавливать через `pip`.

WSGI можно представить так:

```text
браузер
  -> HTTP-запрос
  -> wsgiref
  -> функция application в server.py
```

`wsgiref` принимает настоящий HTTP-запрос и вызывает нашу функцию:

```python
application(environ, start_response)
```

## 7. Что такое `application`

Внизу `server.py` сервер запускается так:

```python
server = make_server(host, port, application)
server.serve_forever()
```

Это значит:

```text
запусти сервер на host:port
каждый новый HTTP-запрос передавай в функцию application
```

Функция `application` - это входная точка нашего веб-приложения.

```python
def application(environ, start_response):
    method = environ["REQUEST_METHOD"]
    path = environ["PATH_INFO"]
```

Каждый раз, когда браузер отправляет запрос, `wsgiref` вызывает `application`.

## 8. Что такое `environ`

`environ` - это обычный словарь Python.

В нем лежит информация о запросе.

Например:

```python
method = environ["REQUEST_METHOD"]
path = environ["PATH_INFO"]
```

Если браузер открыл главную страницу, там будет примерно так:

```python
environ["REQUEST_METHOD"] == "GET"
environ["PATH_INFO"] == "/"
```

Если браузер отправил имя игрока:

```python
environ["REQUEST_METHOD"] == "POST"
environ["PATH_INFO"] == "/api/start"
```

Еще важные поля:

| Поле | Что значит |
| --- | --- |
| `REQUEST_METHOD` | Метод запроса: `GET`, `POST`, `HEAD` |
| `PATH_INFO` | Путь URL: `/`, `/api/start`, `/api/answer` |
| `CONTENT_LENGTH` | Размер тела запроса в байтах |
| `wsgi.input` | Поток, из которого можно прочитать тело запроса |

## 9. Как сервер читает JSON из запроса

Когда браузер отправляет `POST /api/start`, в теле запроса лежит JSON:

```json
{"name": "Аня"}
```

В `server.py` это читает функция `read_json`.

Сначала сервер узнает длину тела:

```python
length_text = environ.get("CONTENT_LENGTH")
```

Потом читает нужное количество байтов:

```python
body_stream = environ["wsgi.input"]
body_bytes = body_stream.read(length)
```

Потом превращает байты в строку:

```python
raw_body = body_bytes.decode("utf-8")
```

Потом превращает JSON-строку в Python-словарь:

```python
data = json.loads(raw_body)
```

После этого можно взять имя:

```python
raw_name = data.get("name", "")
```

## 10. Что такое `start_response`

`start_response` - это функция, которую дает `wsgiref`.

Наша программа вызывает ее, когда готова начать HTTP-ответ.

Пример из `send_bytes`:

```python
start_response(status, headers)
```

Туда передаются:

- статус: например `"200 OK"`;
- заголовки: например `Content-Type`.

После этого функция возвращает тело ответа:

```python
return [body]
```

Почему список?

Так устроен WSGI: тело ответа нужно вернуть как набор частей. В нашем проекте часть одна, поэтому список содержит один элемент.

```python
[body]
```

## 11. Как сервер выбирает нужное действие

В `application` есть явный роутинг:

```python
if read_request == True and path == "/":
    ...
elif method == "POST" and path == "/api/start":
    ...
elif method == "POST" and path == "/api/answer":
    ...
```

Это читается так:

```text
если пришел GET /, отдай index.html
если пришел POST /api/start, начни игру
если пришел POST /api/answer, проверь ответ
```

Роутинг - это выбор Python-кода по URL.

## 12. Путь запроса целиком

Когда игрок нажимает "Старт", путь такой:

```text
1. Игрок нажал кнопку в браузере.
2. JavaScript отправил POST /api/start.
3. wsgiref принял HTTP-запрос.
4. wsgiref вызвал application(environ, start_response).
5. application увидела method == "POST" и path == "/api/start".
6. application вызвала handle_start(start_response, environ).
7. handle_start прочитала JSON через read_json(environ).
8. Python создал игру и первый пример.
9. send_json превратил словарь Python в JSON.
10. send_bytes отправил статус, заголовки и тело ответа.
11. Браузер получил JSON и обновил экран.
```

## 13. Почему это низкоуровневые детали

В этом проекте видны детали, которые фреймворки обычно скрывают:

| Деталь | Что это |
| --- | --- |
| `wsgiref` | Маленький учебный HTTP-сервер из стандартной библиотеки |
| `environ` | Словарь с информацией о запросе |
| `start_response` | Функция для отправки статуса и заголовков |
| Байты | Формат данных, который реально передается по сети |
| Заголовки | Служебная информация HTTP-ответа |

FastAPI, Flask и другие фреймворки часто убирают это из учебного кода.

Тогда можно писать ближе к смыслу приложения:

```text
URL -> функция -> словарь или файл
```

В основной ветке эти детали оставлены специально, чтобы показать, что скрывается под более удобными инструментами.

## 14. Что можно не понимать сразу

Для первого урока не нужно идеально понимать:

- все поля `environ`;
- все HTTP-заголовки;
- почему WSGI возвращает именно список байтов;
- как устроены сокеты;
- как работает HTTPS.

Достаточно понять главную цепочку:

```text
браузер отправляет HTTP-запрос
wsgiref превращает его в environ
application выбирает ветку по method и path
server.py возвращает статус, заголовки и тело ответа
браузер получает ответ и меняет экран
```
