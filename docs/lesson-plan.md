# План объяснения на уроке

## Цель урока

Показать ученику, что веб-приложение состоит из нескольких ролей:

- браузер показывает интерфейс;
- JavaScript отправляет действия игрока на сервер;
- Python-сервер принимает URL-запросы и выполняет правила игры;
- текстовый файл хранит общий счет;
- домен и внешний веб-сервер помогают открыть приложение из интернета.

Не цель урока: подробно изучать CSS, Docker, HTTPS, DNS или промышленную безопасность. Их можно назвать как отдельные слои и не уходить в детали.

## Порядок занятия

1. Открыть `docs/scheme.md` и пройти путь запроса от пользователя до файла.
2. Открыть `docs/http-data-exchange.md` и объяснить HTTP как обмен данными между клиентом и сервером на практических примерах.
3. Открыть `docs/http-wsgi-basics.md` и объяснить, как общий HTTP-обмен попадает в низкоуровневый Python-код через `wsgiref`, `environ`, `start_response`, байты и заголовки.
4. Открыть `static/index.html` и показать, что HTML - это заготовка экрана.
5. Открыть `static/game.js` и показать только крупные действия: старт, ответ, таймер, обновление счета.
6. Открыть `docs/js-client-server-examples.md` и разобрать пары: `fetch(...)` на клиенте и обработчик в `server.py`.
7. Открыть `server.py` и найти функцию `application`.
8. Разобрать таблицу URL из `README.md`.
9. Запустить игру локально командой `python3 server.py`.
10. Ввести имя, решить пару примеров, открыть `data/scores.txt`.
11. Объяснить, что на домене работает тот же проект, только перед ним стоит Caddy.

## Главная часть для ученика

Самое важное место в коде:

```python
if method == "GET" and path == "/":
    ...
elif method == "POST" and path == "/api/start":
    ...
elif method == "POST" and path == "/api/answer":
    ...
```

Это показывает идею веб-сервера без декораторов:

```text
пришел URL -> сервер выбрал ветку кода -> вернул ответ
```

## Что можно дать как маленькое задание

- Добавить деление без остатка.
- Увеличить время игры до 90 секунд через `GAME_SECONDS`.
- Сделать за правильный ответ `+10` очков вместо `+1`.
- Добавить отдельную строку "последний правильный ответ".
- Поменять формат файла счета и обсудить, почему это опасно делать неаккуратно.

## Где остановиться

Если ученик еще не знает классов, не надо разбирать внутренности `wsgiref`. Достаточно сказать:

```text
make_server - готовая функция из стандартной библиотеки Python.
Она умеет слушать порт и передавать запросы в нашу функцию application.
```
