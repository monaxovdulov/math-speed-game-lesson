# Схема ролей

## Короткая схема

```text
Пользователь
  -> Браузер
  -> Домен: питоны-батоны.рф
  -> DNS
  -> IP сервера: 46.226.165.215
  -> Caddy: внешний веб-сервер HTTPS
  -> Python-сервер: math-speed-game-lesson:8000
  -> server.py
  -> data/scores.txt
```

## Роли по уровням

| Уровень | Кто это | Что делает |
| --- | --- | --- |
| Пользователь | Человек перед экраном | Вводит имя, решает примеры, нажимает кнопки |
| Браузер | Chrome, Firefox, Safari или другой браузер | Загружает HTML, CSS и JS, показывает интерфейс, отправляет запросы на сервер |
| HTML | `static/index.html` | Описывает элементы страницы: поле имени, пример, поле ответа, таблицу счета |
| CSS | `static/style.css` | Делает страницу аккуратной: размеры, цвета, отступы |
| JavaScript | `static/game.js` | Запускает таймер, отправляет имя и ответы на сервер, обновляет экран |
| Домен | `питоны-батоны.рф` | Человеческое имя сайта, которое удобно помнить |
| DNS | Система доменных имен | Переводит домен в IP-адрес сервера |
| IP-адрес | `46.226.165.215` | Сетевой адрес машины, где запущен сайт |
| Caddy | Внешний веб-сервер | Принимает HTTPS-запросы с интернета и передает их Python-серверу |
| Python-сервер | `server.py` | Понимает URL, генерирует примеры, проверяет ответы, считает очки |
| Память сервера | Словари `GAMES` и `PROBLEMS` | Временно хранит активную игру и правильные ответы |
| Текстовый файл | `data/scores.txt` | Постоянно хранит общий счет игроков |

## Как запрос проходит через систему

1. Игрок открывает `https://питоны-батоны.рф/`.
2. Браузер спрашивает DNS: какой IP у этого домена?
3. DNS возвращает `46.226.165.215`.
4. Браузер отправляет HTTPS-запрос на этот IP.
5. Caddy принимает защищенный запрос.
6. Caddy передает запрос в контейнер `math-speed-game-lesson` на порт `8000`.
7. FastAPI смотрит на URL и вызывает нужную функцию из `server.py`.
8. Если URL `/`, сервер отдает `index.html`.
9. Если URL `/api/answer`, сервер проверяет ответ и возвращает JSON.
10. Когда игра заканчивается, сервер записывает счет в `data/scores.txt`.

## Почему есть два сервера

В проекте есть два серверных уровня:

| Уровень | Зачем нужен |
| --- | --- |
| Caddy | Стоит снаружи, работает с HTTPS и доменом |
| Python-сервер | Стоит внутри, знает правила игры |

Caddy не знает правил игры. Python-сервер не занимается сертификатами HTTPS. Каждый уровень делает свою работу.
