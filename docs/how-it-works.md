# Как работает приложение

## Главная идея

Приложение состоит из двух частей:

- клиент - код, который работает в браузере игрока;
- сервер - код Python, который работает на машине в интернете.

Клиент отвечает за экран и действия игрока. Сервер отвечает за правила игры и общий счет.

## Что происходит при открытии сайта

Игрок открывает адрес:

```text
https://питоны-батоны.рф/
```

Браузер просит у сервера страницу. В `server.py` это попадает в функцию `application`.

Сервер видит:

```text
method = GET
path = /
```

И выполняет эту ветку:

```python
if method == "GET" and path == "/":
    return send_file(start_response, STATIC_DIR / "index.html", "text/html; charset=utf-8")
```

Так браузер получает HTML-страницу. Потом браузер отдельно запрашивает `/style.css` и `/game.js`.

## Что происходит при старте игры

Игрок вводит имя и нажимает "Старт".

JavaScript отправляет на сервер запрос:

```text
POST /api/start
```

В теле запроса отправляется JSON:

```json
{
  "name": "Аня"
}
```

Сервер делает четыре вещи:

1. Чистит имя игрока, чтобы в нем не было переносов строк и разделителя `|`.
2. Создает случайный `game_id`.
3. Запоминает игру в словаре `GAMES`.
4. Создает первый математический пример.

Пример ответа сервера:

```json
{
  "game_id": "abc123",
  "name": "Аня",
  "seconds": 60,
  "score": 0,
  "problem": {
    "id": "p1",
    "text": "7 * 8"
  }
}
```

Важно: правильный ответ сервер не отправляет заранее. Он хранится на сервере в словаре `PROBLEMS`.

## Что происходит при ответе

Игрок вводит ответ и нажимает "Ответить".

Браузер отправляет:

```text
POST /api/answer
```

Тело запроса:

```json
{
  "game_id": "abc123",
  "problem_id": "p1",
  "answer": "56"
}
```

Сервер:

1. Находит игру по `game_id`.
2. Находит пример по `problem_id`.
3. Проверяет, не закончилось ли время.
4. Превращает ответ игрока в число.
5. Сравнивает число с правильным ответом.
6. Если ответ верный, увеличивает счет игры на `1`.
7. Возвращает новый пример.

## Что происходит при окончании игры

Когда таймер дошел до нуля, браузер отправляет:

```text
POST /api/finish
```

Сервер берет счет текущей попытки и добавляет его в общий счет игрока в файле:

```text
data/scores.txt
```

Если файл был таким:

```text
Аня|12
```

И Аня набрала еще `5`, файл станет таким:

```text
Аня|17
```

## Почему часть данных в памяти, а часть в файле

Активная игра хранится в памяти сервера:

- `GAMES` - текущие игры;
- `PROBLEMS` - правильные ответы для текущих примеров.

Это временные данные. Если сервер перезапустить, активная незаконченная игра пропадет.

Общий счет хранится в текстовом файле. Это постоянные данные. Если сервер перезапустить, счет останется.

## Почему это не база данных

База данных здесь не используется. Сервер сам читает и пишет обычный текстовый файл:

```python
file = open(SCORES_FILE, "r", encoding="utf-8")
text = file.read()
file.close()

file = open(SCORES_FILE, "w", encoding="utf-8")
file.write(text)
file.close()
```

Для первого веб-приложения это полезно: ученик видит, что сервер может сохранять данные без магии.

## Где находится роутинг

Роутинг находится в функции `application` в `server.py`.

Если непонятны слова `wsgiref`, `environ`, `start_response`, заголовки или байты, сначала стоит прочитать `docs/http-wsgi-basics.md`.

Пример:

```python
elif method == "POST" and path == "/api/answer":
    data = read_json(environ)
    ...
```

Это читается так:

```text
если пришел POST-запрос на /api/answer,
то прочитай JSON и проверь ответ игрока
```

## Что такое JSON в этом проекте

JSON - это текстовый формат для передачи данных между браузером и сервером.

Python-словарь:

```python
{"name": "Аня", "score": 5}
```

В JSON выглядит почти так же:

```json
{"name": "Аня", "score": 5}
```

Поэтому JSON удобно объяснять после словарей Python.
